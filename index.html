How TrueKanban works

TrueKanban is intentionally opinionated about the essentials: define workflow, manage flow in-progress, and improve using real signals (WIP, throughput, cycle time, and work item age). 

team@truekanban:~$ start kanban system

┌─ Definition of Workflow (DoW) ───────────────────────────────┐
1. DEFINE → Work item types, start/finish points, states, policies
2. VISUALIZE → Board shows WIP limits, pull signals, SLE, blockers
3. MANAGE → Daily flow review: aging, blocked work, WIP breaches
4. MEASURE → WIP • Throughput • Cycle Time • Work Item Age
5. IMPROVE → Change policies / right-size work / remove blockers
└──────────────────────────────────────────────────────────────┘

FAQ

What makes TrueKanban different from “Kanban boards” in other tools?

Many tools offer columns and cards. TrueKanban is built around the Kanban system essentials: explicit workflow policies, WIP control, aging + SLE management, and the core flow metrics.

Do you support Scrum too?

The focus is “pure Kanban.” If you run Scrum, you can still use TrueKanban for flow tracking and policies, but we won’t force Scrum artifacts or timeboxes.

How do you handle SLEs if we don’t have historical data yet?

Start with a reasonable best-guess SLE, then refine once you have enough cycle time history. The tool will help you visualize risk while you learn.

Is forecasting deterministic?

No—forecasting is probabilistic and based on historical performance. You’ll get ranges and confidence, not made-up certainty.
